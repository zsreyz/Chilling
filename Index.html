3What is HTTP vs HTTPS
4What is a domain name (like youtube.com)
5What is web hosting
6What is DNS (converts domain to IP address)

4Paragraph: p tag
5Links: a href=
6Images: img src=
7Lists: ul, ol, li
8Forms: form, input, button, textarea
9Divs and Spans (containers)
10Semantic HTML: header, nav, main, footer

3Operators: +, -, *, /, ===, !==, &&, ||

User accounts, data storage, payments — none of that works without a backend. This turns your demo into a real product people sign up for.
🏆
PHASE MILESTONE
Working API with user signup, login, and database. Connected to your React frontend.
Node.js lets JavaScript run on a server. Express makes building APIs easy. This is your startup's backend — receives requests, does logic, sends back data.
App calls your server → Express receives it → Does logic → Returns JSON data
1What is Node.js (JS outside browser)
2npm — install packages (like App Store for code)
3Express routes: GET, POST, PUT, DELETE
4req.body, req.params, req.query
5res.json() — sending data back
6Middleware concept
7Environment variables (.env file)

MongoDB stores data as JSON documents. Flexible, beginner-friendly, free cloud version. This is where all your users, posts, and startup data lives permanently.
User signs up → Express saves to MongoDB → Data lives forever → User logs in next day → Data retrieved
1SQL vs NoSQL (MongoDB is NoSQL)
2MongoDB Atlas — free cloud database
3Mongoose — connects Node to MongoDB
4Schema and Model
5CRUD: Create, Read, Update, Delete

Firebase = Google's ready-made backend. Auth, database, file storage — all free, all instant. Many startups launch on Firebase first to move fast.
Connect Firebase SDK → Use built-in functions → No server to build
1Firebase Authentication (Google login, email/pass)
2Firestore Database (real-time)

🧠 Founder Mindset

What separates builders from dreamers

⚡

You don't need to know everything. You need to know ENOUGH to build your idea and figure out the rest as you go.

🎯

Build while you learn. Every phase ends with something REAL you made. Projects teach you 10x more than watching videos.

🔥

3-4 hours/day = 1,000+ hours in a year. That's more than enough to go from zero to launching a real startup.

🚫

Don't wait till you're 'ready'. Ship ugly, learn fast, improve always. Every successful startup launched with an embarrassing first version.

💡

Your first startup idea will probably change. The coding skills you build won't. Learn the fundamentals deeply.
